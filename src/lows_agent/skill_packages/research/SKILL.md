# 跨法域法律研究

## 1. 确定研究边界

在搜索前明确以下信息：

1. **法域**：国家或地区；必要时继续区分联邦与州、中央与地方。
2. **争点**：把用户问题改写成可检索的法律问题，不改变事实。
3. **目标时间**：确认研究当前有效法律还是历史时点的法律。
4. **资料类型**：立法、行政规则、司法裁判、官方指引或二手解释。

法域或目标时间会改变结论且无法合理判断时，先询问用户；可以继续检索时，明确写出临时假设。

## 2. 按来源层级检索

必须优先检索第一手官方来源：

1. 官方立法数据库、议会或政府公报；
2. 官方行政法规、规章和监管机构材料；
3. 官方法院、判例或司法解释数据库；
4. 政府发布的指南、问答或执法文件；
5. 学术、专业数据库和可靠二手资料；
6. 普通网页仅用于发现线索。

中国法先使用国家法律法规数据库、全国人大和司法部；案例库内容只能作为案例或实务参考。其他法域按 `references/jurisdiction_sources.md` 选择官方域名。

## 3. 核验原文

在作出确定性陈述前读取候选原文，并核对：

- 标题、发布机关、文号或案件编号；
- 颁布、修订、生效和废止日期；
- 法律层级、适用地域和适用对象；
- 条文上下文、定义、例外和过渡条款；
- 案件的法院层级、裁判日期、程序状态和先例效力。

搜索摘要不是原文。无法读取原文时，可以把摘要作为待核实线索，但必须标记“尚未完成原文核验”。

## 4. 处理冲突

不同来源冲突时，优先采用权限更高、时间更近且直接发布该文件的官方来源。不要用后发布的网页解读自动覆盖仍有效的成文法，也不要把案例结论泛化到不同事实或法域。

## 5. 输出

先回答争点，再列依据。每项关键依据包含：

- 资料标题；
- 发布机关或法院；
- 资料类型和效力层级；
- 适用法域与目标时间；
- 支撑结论的内容；
- 官方原文 URL；
- 无法确认的限制。

将法律规则、案例适用和风险判断分开表达。不得编造条文、案件、文号、日期、引语或效力状态。
